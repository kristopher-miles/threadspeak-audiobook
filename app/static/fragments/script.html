Generate Script

Now that your script is uploaded, use the button below to process the entire book script. This script will be identical to your original book. (For a looser audioplay translation, toggle legacy mode in the settings.) This process uses a text LLM and sends hundreds or even thousands of requests, depending on the length of your book. It is intended to use a locally-hosted, “weak” model with tool-calling capability. If you use a cloud model, choose a cheap or free endpoint. Even with a powerful GPU, this can take many hours to complete.
Script Operations
Sends the book to your LLM to split it into annotated chunks with speaker labels and voice directions. Runs a second LLM pass to fix speaker misattributions or formatting issues. Only needed if the generated script has problems. Compares the finished script to the original uploaded source, ignores punctuation/case/spacing, and records only the invalid text spans that need regeneration. Regenerates each invalid block from the original source, patches the live script, then reruns the sanity check to confirm parity or report what remains.
Splits the source document into individual paragraphs and prepares them for dialogue analysis. Analyzes each paragraph to identify speakers and annotate dialogue lines with voice assignments. Analyzes narration paragraphs to identify emotional mood and delivery style for each one. Builds the audiobook script from processed paragraphs, assigning speakers, moods, and splitting into lines.
